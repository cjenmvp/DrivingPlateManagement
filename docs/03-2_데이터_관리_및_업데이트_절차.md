# 데이터 관리 및 업데이트 절차 가이드

본 문서는 새로운 드라이빙 플레이트 소스가 추가될 때 수행해야 하는 표준 작업 절차를 정의합니다.

## 1. 신규 소스 입고 절차

### **1.1 가이딩 명칭 결정 (Naming)**
촬영 로그를 바탕으로 `표준 명명 규칙(v2.0)`에 따라 폴더명을 생성합니다.
- 예: `260316_D01_24p_V-log_nclc_새로운 촬영지`

### **1.2 폴더 생성 및 마스터 복사**
- `Z:\2_DrivingPlate_Library_nclc\` 경로에 위에서 결정한 이름으로 폴더를 생성합니다.
- 카메라별 원본 파일(.mov)을 복사하고, 파일명을 `[카메라ID]_[폴더명].mov` 형식으로 변경합니다.

### **1.3 프록시(Proxy) 생성**
- 편집 및 확인 용이성을 위해 프록시 파일을 생성합니다.
- **규격:** H.264 / 1920x1080 / Low Bitrate
- **파일명:** `[카메라ID]_[폴더명]_proxy.mov`
- **저장 위치:** `Z:\2_DrivingPlate_Library_nclc\Proxy\`

## 2. 인벤토리 업데이트 (Sync)

새로운 폴더가 추가되면 다음 문서들을 반드시 업데이트해야 합니다.

1. **Obsidian 인벤토리**: `드라이빙소스운영방안\02_전체_라이브러리_인벤토리.md`에 새로운 항목 추가.
2. **엑셀 시트용 CSV**: `LIBRARY_FOR_SHEET.csv` 파일에 새로운 데이터 행 추가.
3. **AI 참조 문서**: `LLM_LIBRARY_REFERENCE.md`에 변경 사항 반영.
