# 06. 커넥트 (Connect - CLI/MCP): 현실 세계의 연결

## 1. 개요
**"툴과 스토리지를 어떻게 하나로 엮을 것인가?"**

이 문서는 VP Stage 운영 프레임워크의 6번째 단계인 **커넥트(Connect)** 에 대한 상세 매뉴얼입니다. 
다양한 오프라인 스토리지 자산(NotchLC 원본영상, 프록시 파일 등), 외부 클라우드 문서(Notion), 그리고 이를 제어하는 AI 에이전트 간의 통신과 제어 권한을 통합하는 연결 고리(Connection)의 구조적 설계와 활용 방안을 다룹니다.

---

## 2. 핵심 연결 요소 (Core Connections)

커넥트 단계는 크게 두 가지 핵심 프로토콜과 API를 중심으로 구축됩니다.

### 2.1 MCP (Model Context Protocol) 기반 연결
AI 에이전트(Antigravity)가 로컬 환경 및 외부 서비스의 데이터에 안전하고 표준화된 방식으로 접근할 수 있게 해주는 핵심 프로토콜입니다.

*   **역할**: 
    *   에이전트에게 로컬 파일 시스템(`X:\`, `Z:\`)의 상태를 실시간으로 읽고 제어할 수 있는 '눈과 손'을 제공합니다.
    *   Notion과 같은 외부 서비스와 연동하여, 사람이 작성한 지침이나 데이터(예: LED 물리적 스펙, 전력량)를 에이전트가 직접 읽어들일 수 있는 통로가 됩니다.
*   **주요 활용 사례**:
    *   **Notion MCP Server 연동**: VP Stage의 최신 도면, 장비 리스트, 운영 매뉴얼 등을 Notion API를 통해 에이전트가 직접 파싱하여 로컬 DB(SQLite)와 동기화합니다.
    *   **Local File System MCP (예정)**: (존재할 경우) 스토리지 내 미디어 파일의 입출고 상태, 메타데이터 추출 요청 등을 에이전트가 수행하도록 돕습니다.

### 2.2 CLI (Command Line Interface) 및 스크립트 기반 제어
MCP를 통해 상황을 파악하고 결정을 내린 에이전트가 실제로 OS 레벨의 제어 명령을 내리는 실행 창구입니다.

*   **역할**: OS 스크립트를 직접 실행하여 단순화, 자동화된 백그라운드 작업을 처리합니다.
*   **주요 활용 사례**:
    *   `mklink` (Windows) 를 활용한 심볼릭 링크(Symbolic Link) 자동 생성으로 무거운 원본 데이터의 복사 시간 및 용량을 절약합니다.
    *   `FFmpeg` / `FFprobe` CLI 커맨드를 에이전트가 직접 실행하여 인제스트된영상의 프록시 자동 변환 및 메타데이터 추출을 수행합니다.
    *   개발된 Python 스크립트(`update_db.py` 등)를 터미널을 통해 실행시켜 데이터베이스 동기화를 트리거합니다.

### 2.3 Web/API 기반 연결 (Supabase/DB)
로컬 데이터베이스 시스템과 사용자가 상호작용하는 웹 기반 UI(또는 라이브러리 매니저) 간의 연결입니다.

*   **역할**: 실시간 혹은 스케줄에 따른 시스템 상태 트래킹을 제공합니다.
*   **주요 활용 사례 (계획중)**: 
    *   로컬 SQLite (또는 Supabase) 실시간 통신을 통해 신규 미디어가 입고되고 프록시 변환이 완료되면 사용자 화면(Web UI)에 즉시 썸네일과 메타데이터가 업데이트됩니다.

---

## 3. 커넥트 설계의 강점

1. **상호 운용성(Interoperability)**: 서로 다른 프로토콜(Notion API, 로컬 SQLite, Windows cmd)의 데이터 형식을 Agent라는 공통의 두뇌를 통해 하나로 연결하고 번역합니다.
2. **자동화의 완성**: 스크립트(CLI) + 통신(API/MCP)이 결합되어 인간의 물리적 개입(파일 복사, 엑셀 기록)을 최소화하는 완전 자동화 파이프라인(Zero-Touch Pipeline)을 구축합니다.
3. **유연한 확장성**: 추후 외부 시스템(예: 새로운 렌더링 엔진 API, 장비 제어 API)이 추가되더라도 MCP 서버의 확장만으로 에이전트가 즉각적으로 새로운 장비를 제어할 수 있습니다.
