# LLM별 세션 재개 지침 (v1.0)

본 문서는 새로운 세션이 시작될 때 각 AI 에이전트(Gemini, Codex, Claude)가 신속하게 컨텍스트를 복구하고 일관된 작업을 수행하기 위한 지침입니다.

---

## 🛠️ 공통 필수 프로토콜
새로운 세션을 시작하는 모든 에이전트는 다음 순서대로 파일을 읽어야 합니다.
1. `gemini.md` (기본 원칙 및 언어 설정)
2. `docs/SESSION_STATE.md` (현재 작업의 가장 최신 상태)
3. `HANDOVER.md` (이전 에이전트로부터의 구체적인 미션)

---

## 🤖 에이전트별 개별 지침

### 1. Antigravity AI (Gemini CLI)
- **핵심 역할**: 파일 시스템 제어 및 OS 레벨 실행.
- **재개 지침**: 
  - `X:` 및 `Z:` 드라이브 마운트 상태를 먼저 확인하십시오.
  - `git status`를 통해 로컬 파일과 원격(GitHub) 간의 싱크를 체크하십시오.
  - 실행 중인 백그라운드 프로세스가 있는지 `Get-Job` 등으로 확인하십시오.

### 2. Codex CLI GPT 5.2
- **핵심 역할**: 코드 정교화 및 DB 쿼리 최적화.
- **재개 지침**: 
  - `data/library.db` (SQLite)의 현재 스키마를 `PRAGMA table_info`로 확인하십시오.
  - `scripts/` 내의 Bash 스크립트가 최신 버전인지 GitHub 기록과 대조하십시오.
  - 이전 세션에서 제안된 로직의 논리적 무결성을 검토하십시오.

### 3. Claude (UX/Context)
- **핵심 역할**: 문서화 및 맥락 분석.
- **재개 지침**: 
  - `docs/` 내의 최신 계획서들을 읽고 사업적/철학적 방향성(Ontology)을 재확인하십시오.
  - 사용자(User)의 최신 피드백이 담긴 세션 로그를 분석하여 UX 개선점을 도출하십시오.
