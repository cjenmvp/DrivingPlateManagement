# 비선호 카메라 각도 정리 계획 (2026-03-16)

본 문서는 드라이빙 플레이트 활용도가 낮은 특정 카메라 각도 파일들을 별도로 분리하여, 실무 작업 시 필요한 소스만 빠르게 식별할 수 있도록 하는 운영 계획을 정리한 것입니다.

---

## 🎯 정리 대상 (제외 각도)
드라이빙 촬영 목적상 활용도가 낮거나 중복되는 아래 4개 카메라 ID를 관리 대상에서 분리합니다.
- **01_**: (예: 천장/수직 각도 등)
- **05_**: (예: 비선호 측면/후방 각도)
- **07_**: (예: 비선호 측면/후방 각도)
- **11_**: (예: 비선호 측면/후방 각도)

---

## 📂 분류 방식 및 구조
각 소스 폴더 내에 하위 폴더를 생성하여 파일을 이동시킵니다.

### 1. 폴더 구조 변경 (안)
```text
[Source_Folder]/
├── 02_...mov (메인 활용 각도)
├── 03_...mov
├── 04_...mov
├── 06_...mov
├── ...
└── _Excluded_Angles/  <-- 신규 생성 (비선호 각도 보관)
    ├── 01_...mov
    ├── 05_...mov
    ├── 07_...mov
    └── 11_...mov
```

### 2. 적용 범위
- **마스터(Z:)**: `Z:\2_DrivingPlate_Library_nclc` 내 모든 폴더
- **프록시(X:)**: `X:\06_DrivingPlate_Library_nclc_proxy` 내 모든 폴더

---

## 🛠️ 자동화 처리 계획
수동 이동 시 실수가 발생할 수 있으므로, 아래 로직의 스크립트를 통해 일괄 처리합니다.

1. 각 드라이브의 모든 소스 폴더를 순회.
2. 각 폴더 내에 `_Excluded_Angles` 폴더가 없으면 생성.
3. 파일명 시작이 `01_`, `05_`, `07_`, `11_`인 파일들을 해당 폴더로 이동.
4. 이동 후 메인 폴더에는 활용도가 높은 8~11개 내외의 파일만 남게 됨.

## ✅ 기대 효과
1. **시인성 확보**: 작업 폴더 진입 시 불필요한 파일이 숨겨져 소스 선택이 빨라짐.
2. **데이터 보존**: 파일을 삭제하는 것이 아니라 하위 폴더로 옮기는 것이므로, 추후 특수 각도가 필요할 때 즉시 대응 가능.
3. **인벤토리 단순화**: 15~17개에 달하는 파일을 10개 내외로 압축하여 관리 효율 증대.
