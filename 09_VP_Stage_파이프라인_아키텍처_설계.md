# VP Stage 드라이빙 플레이트 파이프라인 아키텍처 설계 (v1.0)

본 문서는 30TB 이상의 방대한 드라이빙 플레이트 데이터를 체계적으로 관리하고, 가상 스튜디오(VP Stage)의 Disguise 미디어서버로 데이터를 빠르고 안전하게 공급하기 위한 **통합 파이프라인 구축 마스터플랜**입니다.

---

## 🏗️ 1. 시스템 아키텍처 (System Architecture)

전체 시스템은 **검색(DB) → 미리보기(Proxy) → 전송(Sync)**의 3단계로 구성됩니다.

### A. 데이터베이스 & 백엔드 (Supabase)
- **플랫폼**: [Supabase](https://supabase.com/) (오픈소스 PostgreSQL 기반 클라우드/로컬 DB)
- **역할**: 
  - 수만 개의 메타데이터(경로, 길이, 코덱, 태그, 시간대 등) 인덱싱.
  - REST API 및 실시간 구독(Realtime) 기능을 통해 프런트엔드 툴에 즉각적인 데이터 제공.
- **장점**: 대용량 파일 시스템을 직접 스캔할 필요 없이 0.1초 만에 원하는 소스 검색 가능.

### B. 관리 및 미리보기 툴 (Library Manager App)
- **프록시(Proxy) 기반 재생**: 무거운 마스터 파일 대신 `X:\` 드라이브에 있는 H.264 프록시 파일을 불러와 부하 없이 실시간 미리보기 제공.
- **개발 환경**: 웹 기반(React/Vue) 대시보드 또는 PC 설치형 앱(Electron, Python PyQt 등) 형태로 구축.

### C. Disguise 동기화 엔진 (Sync & Deploy)
- **작동 방식**: 관리 툴에서 특정 소스를 선택(Pick)하면, 툴이 알아서 `Z:\` 드라이브의 NotchLC 원본 파일을 찾아냅니다.
- **전송 옵션**:
  1. **심볼릭 링크 (Symbolic Link)**: 스토리지가 동일 네트워크에 마운트되어 있을 경우, 용량 낭비 없이 디렉터 머신에 가상 파일 경로만 생성.
  2. **고속 동기화 (Data Sync)**: 디렉터 머신의 로컬 SSD로 보내야 할 경우 `Robocopy`나 `rsync` 기반의 백그라운드 고속 복사 스크립트 실행.

## 🔄 2. 제작 워크플로우 (Workflow Pipeline)

1. **Ingest (입고)**
   - 신규 소스 촬영 → `Z:\` 드라이브 입고 및 NotchLC 변환.
   - 자동화 스크립트가 `FFprobe`로 메타데이터를 추출하여 **Supabase DB에 Insert**.
   - AME(Adobe Media Encoder)를 통해 `X:\` 드라이브에 프록시 자동 생성.

2. **Curation (검색 및 기획)**
   - 작업자가 프런트엔드 툴에 접속하여 Supabase DB를 통해 "야간, 강남역, 60fps" 조건 검색.
   - 프록시 영상으로 빠르게 컷 확인.

3. **Deploy (송출 준비)**
   - 선택된 클립들의 플레이리스트를 구성하고 'Disguise로 보내기' 클릭.
   - 툴이 원본(NotchLC)의 심볼릭 링크를 디렉터 머신의 특정 프로젝트 폴더(`d3 projects/Video`)에 자동 생성하여 즉각적인 맵핑 준비 완료.
