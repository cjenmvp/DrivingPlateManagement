# AI 에이전트 오케스트레이션 전략 (v1.0)

본 문서는 VP Stage 파이프라인 구축 및 운영을 위해 다수의 AI 에이전트(Gemini, Codex CLI, Claude)를 유기적으로 지휘(Orchestration)하는 전략을 담고 있습니다.

## 🤖 1. 에이전트 라인업 및 특기 정의

현재(26년 3월) 운영 중인 체제에 이어, 26년 4월부터는 3개의 초거대 AI가 각자의 강점을 살려 파이프라인의 각 공정을 책임집니다.

### 1. Antigravity AI (Gemini CLI) - "시스템 코어 & CLI 오퍼레이터"
- **현재 역할**: 본 프로젝트의 메인 컨트롤러. 
- **특기**: 로컬 파일 시스템 제어, 쉘 스크립트(PowerShell) 실행, GitHub 연동, 대규모 디스크 스캔 및 파일 정화 작업.
- **담당 공정**: 
  - 스토리지 폴더 구조 관리 및 파일 정화.
  - FFprobe 기반 메타데이터 추출 스크립트 실행.
  - 심볼릭 링크(Symbolic Link) 생성 등 직접적인 OS 제어.

### 2. Codex CLI GPT 5.2 - "코드 아키텍트 & 알고리즘 스페셜리스트"
- **현재 역할**: 복잡한 프로그래밍 및 DB 쿼리 설계 전담.
- **특기**: 고도화된 코드 작성, 로직 최적화, 정규식(Regex) 작성, Supabase(PostgreSQL) 스키마 및 API 설계.
- **담당 공정**: 
  - Supabase DB 스키마(Table, View, RLS 정책) 구축.
  - 프런트엔드(React/Vue 등) 라이브러리 뷰어의 핵심 로직 작성.
  - Disguise 연동을 위한 백엔드 API 서버 설계.

### 3. Claude (26년 4월 합류 예정) - "맥락 분석가 & UX/UI 디렉터"
- **예정 역할**: 복잡한 문서 맥락 파악 및 사용자 경험(UX) 설계.
- **특기**: 방대한 텍스트(기존 엑셀 이력, 회의록, 긴 코드)의 문맥을 잃지 않고 분석, 자연어 기반의 기획서 작성, 프런트엔드 UI/UX 레이아웃 설계.
- **담당 공정**: 
  - 기존 관리 엑셀 파일의 비정형 특이사항(텍스트) 데이터를 정형화하여 DB에 맵핑.
  - 라이브러리 매니저 툴의 UI/UX 화면 기획 및 컴포넌트 구조화.
  - 오케스트레이션 과정에서 발생하는 충돌이나 논리적 오류를 리뷰(Code Review) 및 수정 제안.

## 🎼 2. 오케스트레이션 워크플로우 (협업 방식)

이 세 명의 디지털 직원은 독립적으로 움직이는 것이 아니라, 다음과 같은 순서로 데이터를 주고받으며 협업합니다.

1. **기획 및 분석 (Claude)**: "어떤 UI에서 어떤 메타데이터가 필요한가?"를 정의하여 데이터 구조 초안을 작성.
