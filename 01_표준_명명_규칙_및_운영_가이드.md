# 드라이빙 플레이트 라이브러리 표준 명명 규칙 (v2.0)

본 문서는 차량 주행 영상(Driving Plates) 라이브러리의 체계적인 관리와 검색 효율성을 위해 정의된 표준 운영 방안입니다. 모든 데이터는 본 규칙에 따라 명명되고 관리되어야 합니다.

## 1. 폴더 명명 규칙 (Folder Naming)

모든 프로젝트 폴더는 아래의 6가지 핵심 메타데이터를 포함하는 표준 형식을 따릅니다.

### **구조: `[날짜]_[코드]_[프레임]_[컬러]_[프로파일]_[장소명]`**
- **예시:** `251104_D09_24p_V-log_nclc_학동 가구거리 동향`

### **필드별 상세 정의**
1. **날짜 (YYMMDD)**: 촬영 일자 6자리 (예: 251104)
2. **코드 (Time Code)**: 시간대 및 시퀀스 식별자
   - `D`: 주간 (Day)
   - `N`: 야간 (Night)
   - `MH`: 매직 아워 (Magic Hour)
   - `M`: 새벽/아침 (Morning)
   - *숫자 조합(D01, N04 등)은 당일 촬영된 동일 시간대의 시퀀스 순번을 의미합니다.*
3. **프레임 (FPS)**: `24p`, `60p` 등 초당 프레임 수
4. **컬러 (Color Space)**: 적용된 로그 프로파일 또는 컬러 규격
   - `709`: Rec.709 (표준)
   - `D-log`: DJI Log
   - `V-log`: Panasonic/Sony Log
5. **프로파일 (Metadata)**: 컬러 태그 또는 압축 규격
   - `nclc`: 특정 컬러 프로파일 유지 태그
6. **장소명**: 구체적인 촬영 장소, 도로명 또는 주행 방향

## 2. 파일 명명 규칙 (File Naming)

폴더 내부의 개별 영상 파일은 카메라 위치 식별자와 폴더명을 결합하여 명명합니다.

### **원본 소스 (Master)**
- **구조:** `[카메라ID]_[폴더명].mov`
- **예시:** `02_251104_D09_24p_V-log_nclc_학동 가구거리 동향.mov`
- **카메라 ID 식별자:**
  - `02`: 전방 메인
  - `03`, `03T`: 전방 측면 / 상단
  - `08`, `09`, `09T`: 후방 / 측면
  - *숫자가 낮을수록 전방, 높을수록 후방을 의미합니다.*

### **프록시 소스 (Proxy)**
- **구조:** `[카메라ID]_[폴더명]_proxy.mov`
- **관리 경로:** `Z:\2_DrivingPlate_Library_nclc\Proxy\` 폴더 내에 별도 관리하거나 해당 폴더 내 `Proxy` 서브폴더에 위치.

## 3. 디렉토리 구조 (Directory Structure)

```text
Z:\2_DrivingPlate_Library_nclc\
├── [날짜]_[코드]_[프레임]_[컬러]_[프로파일]_[장소명]/  (마스터 폴더)
│   ├── 02_[폴더명].mov
│   ├── 03_[폴더명].mov
│   └── ...
├── Proxy/                                          (전체 프록시 통합 관리)
├── aset/                                           (참조 데이터 A)
└── bset/                                           (참조 데이터 B)
```
